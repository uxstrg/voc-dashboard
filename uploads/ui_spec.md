## 0. 설계 철학

### 이 서비스가 답하는 질문
> "지금 어느 영역에서 고객 기대치가 이탈하고 있는가?"

### AI의 역할 원칙
AI는 전체를 요약하지 않는다.
각 도메인 담당자에게 **지금 당장 의미 있는 신호를 표면화**하는 것이 역할이다.

### 핵심 설계 원칙
- 숫자보다 상태 신호가 먼저 읽혀야 한다
- 도메인 카드가 화면의 주인공이다. 전체 진단은 맥락이다
- 분석된 데이터와 날 것의 목소리가 공존해야 이 서비스의 정체성이 느껴진다
- 사용자가 클릭 1회로 자기 영역의 문제에 도달할 수 있어야 한다

### 사용자와 사용 목적
| 사용자 | 이 화면을 여는 이유 | 원하는 답 |
|--------|-------------------|-----------|
| UX 기획자 | "이번 주 UX에서 뭐가 새로 터졌어?" | 어떤 속성이 나빠졌고 어떤 VoC가 반복되는지 |
| 기술/개발팀 | "지금 시스템 이슈 있어?" | 앱 크래시·로딩 클레임이 몇 건이고 얼마나 심각한지 |
| 전략/상품팀 | "가격·혜택 불만 요즘 어때?" | 가격합리성·타겟적합성 부정 패턴 현황 |
| CS/운영팀 | "상담 클레임 튀는 거 있어?" | 지원 신속성·처리 정확성·투명성 이슈 현황 |
| 경영진/전사 | "전체 브랜드 건강 상태는?" | 도메인별 위험 신호 한눈에 파악 |

## 3. 화면 명세

### 3.1 종합 진단 현황 (`/`)

**목적**: 전체 브랜드 상태를 빠르게 파악하고, 담당 도메인의 이상 신호에 즉시 도달

---

#### 전체 화면 구조

```
[헤더] GNB 1줄 — 로고 + 탭(종합진단/VoC피드/가이드) + 전체 지표

[A] 진단 헤더
  ├── 상단: 분석 기간 · VoC 수 · 이슈 수 · 긍정 반응 (메타 정보)
  └── 하단 3칸: 종합 점수 | 이번 주 전체 패턴 | 지금 당장

[B] 도메인 카드 4개  ← 화면의 히어로
    └── [C] 도메인 상세 패널 (카드 클릭 시 아코디언)
          ├── [C-1] 도메인 속성 레이더 + AI 트리거
          ├── [C-2] VoC 이슈 테이블 (상위 7건)
          └── [C-3] 이번 주 대표 보이스 (원문 1~2개)

---- 스크롤 구간 ----

[D] 채널별 감성 분포 + 도메인 점수 추이 (4주)
```

---

#### [헤더] GNB (1줄)

| 위치 | 요소 | 명세 |
|------|------|------|
| 좌측 | Simply U+ 로고 | 서비스명 + 부제 |
| 중앙 | 탭 네비게이션 | 종합 진단 `/` · VoC 피드 `/voc-feed` · 분석 기준 가이드 `/guide` |
| 우측 | 전체 지표 | 총 VoC 수 · 분석 이슈 수 · 기준 날짜 |

---

#### [A] 진단 헤더

위치: GNB 바로 아래, 전체 너비

**상단 메타 행**
```
분석 기간: YYYY.MM.DD – MM.DD · N일  |  수집 VoC: N건  |  분해 이슈: N건  |  긍정 반응: N%
```
배경: 연회색(`#F9FAFB`).

**하단 인사이트 행 (3칸)**

| 칸 | 내용 | 역할 |
|----|------|------|
| 좌 (종합 점수) | 4개 도메인 가중 평균 점수 + 상태 뱃지 | 전체가 지금 어느 수준인지 |
| 중 (이번 주 전체 패턴) | 도메인을 가로지르는 연쇄 패턴 + 행동 힌트 | 카드에서 안 보이는 전체 관통 신호 |
| 우 (지금 당장) | 도메인 뱃지 + 긴급 현상 문장 최대 3건 | 즉각 대응 필요 신호 |

**좌칸 — 종합 점수**
- 점수 = 4개 도메인 점수 가중 평균 (analysis_logic.md 11항 참고)
- 상태 뱃지: 양호(≥75) / 보통(60~74) / 주의(45~59) / 위험(≤44)

**중앙칸 — 이번 주 전체 패턴**
- 도메인 간 연쇄 패턴 표현. 2줄 이내
- 텍스트 규칙 → `output_design.md` 2.0항

**우측칸 — 지금 당장**
- 최대 3건, 심각한 순 정렬
- 클릭 시 해당 도메인 카드 아코디언 자동 펼침
- 텍스트 규칙 → `output_design.md` 2.0항

---

#### [B] 도메인 카드 4개

레이아웃: 4개 가로 균등 배치

카드 내 정보 순서:
```
[도메인명]                         [상태 뱃지: 위험/주의/보통/양호]
[점수]점  [▼Npt / ▲Npt]
[바 게이지]
[#핵심속성1↓]  [#핵심속성2↓]
[AI 트리거 한 줄]
[↑N  ↓N]                                          [▼ 탐색]
```

| 요소 | 명세 |
|------|------|
| 상태 뱃지 | 위험(≤44)/주의(45~59)/보통(60~74)/양호(≥75), 우측 상단 |
| 점수 | 0~100, 28px bold |
| 등락 표시 | ▼Npt Red / ▲Npt Green, 전주 대비 |
| 바 게이지 | 점수에 비례, 도메인 컬러 |
| 속성 태그 | 부정 비율 높은 속성 1~2개, `#태그명↓` 형식 |
| AI 트리거 | 15자 이내 명사형 (output_design.md 2.2항) |
| VoC 카운트 | 긍정↑N 부정↓N |
| 탐색 버튼 | 클릭 시 [C] 아코디언 펼침 |

---

#### [C] 도메인 상세 패널 (아코디언)

위치: 클릭된 카드 바로 아래, 전체 너비
레이아웃: 좌(40%) + 우(60%) 2단

**[C-1] 좌: 도메인 속성 레이더 + AI 트리거**
- 해당 도메인 속성만 표시 (전략 7개 / UX 15개 / 기술 3개 / 운영 4개)
- 2개 레이어: 기대치(점선) + 실제(실선, 도메인 컬러)
- 하단: AI 트리거 박스 1줄 (15자 이내, output_design.md 2.2항)

**[C-2] 우: VoC 이슈 테이블**

컬럼: 우선순위(High/Mid/Low) / 속성명 / 감성 뱃지 / 핵심 내용 한 줄 / 펼침 토글
기본 정렬: 부정 우선 → High 우선순위 순
표시: 상위 7건 + `전체 보기 →` (/voc-feed 이동)

행 클릭 시 인라인 펼침:
```
원본 VoC 전문 (날 것 그대로)
출처: [채널명]  [서비스명]
액션 힌트: [동사형 실무 지시] (output_design.md 2.4항)
```

**[C-3] 하단: 이번 주 대표 보이스**

| 항목 | 명세 |
|------|------|
| 표시 건수 | 1~2개, 부정+High 기준 선별 |
| 형식 | 큰따옴표 인용 스타일, 분석 태그 없이 원문만 |
| 클릭 | /voc-feed 해당 항목으로 이동 |

---

#### [D] 스크롤 구간 — 채널 + 트렌드

레이아웃: 좌(채널 35%) + 우(트렌드 65%) 2단

**[D-1] 채널별 감성 분포**
- iOS / AOS / Threads / X / CS 설문
- 긍정·부정·중립 비율 수평 바 차트 + 건수

**[D-2] 도메인 점수 추이**
- 최근 4주, 4개 도메인 라인 차트
- 하단: 도메인별 현재 점수 + 전주 대비 등락

### 3.2 VoC 원문 피드 (`/voc-feed`)

**목적**: 진단 완료된 VoC 원문 전체 열람  
**서비스 정체성 핵심 화면**: 가공 전 원문이 주인공

#### 상단 필터 바
- 출처: iOS / AOS / Threads / X / 커뮤니티
- 도메인: 전체 / 전략 / UX / 운영 / 기술
- 원문 검색

#### VoC 카드 리스트
- 원문 텍스트 전문 (날 것 강조)
- 출처 뱃지 + 수집 시각
- 분해 이슈 수 / 주요 도메인 뱃지 / 우선순위(High) 표시

#### 개별 VoC 드릴다운 (카드 클릭 또는 슬라이드 패널)
- 원문 전체
- 분해된 이슈 목록 (속성 / 감성 / Gap / 상태)
- AI 종합 코멘트
- 액션 힌트

**목적**: 스코어링 기준 및 29개 속성 참고

- 스코어링 공식 (Score_reality, Gap, 상태 판정)
- 29개 속성 목록 (도메인별 탭 + 검색)

## 4. 디자인 원칙

### 테마
- 라이트 모드 전용
- 배경: 흰색(`#FFFFFF`) / 연회색(`#F9FAFB`)
- 다크 모드 미지원

### 스캐너빌리티
- 3초 안에 핵심 신호 파악 가능해야 함
- 색상 = 의미 (위험 Red / 주의 Amber / 양호 Green 일관)
- 상태 뱃지가 점수보다 먼저 읽혀야 함

### 정보 계층
| 레벨 | 내용 | 진입 방법 |
|------|------|-----------|
| 0 | 긴급 신호 | 진단 헤더 우측칸 자동 표시 |
| 1 | 전체 맥락 스캔 | 진입 시 기본 |
| 2 | 도메인 세부 | 카드 클릭 (아코디언) |
| 3 | 개별 이슈 | 테이블 행 클릭 |
| 4 | 원문 탐색 | /voc-feed 이동 |

### 날 것의 목소리 보존
- /voc-feed는 원문이 주인공
- [C-3] 대표 보이스는 분석 없이 원문만
- "유플 왜 이래?" 같은 말이 살아있어야 이 서비스의 정체성이 느껴짐

### Phase 2 이후 구현 (현재 범위 외 — 구현하지 말 것)

| 항목 | 이유 | 대체 방식 |
|------|------|-----------|
| VoC 수동 입력 + 실시간 AI 진단 | MVP는 정적 사이트 | 진단 트랙에서 별도 수행 |
| 진단 결과 승인/수정 워크플로우 | MVP에 불필요 | Phase 2 |
| 브랜드 인식 가치 레이더 | 도메인 점수와 시각적 중복 | data_schema.md에 구조 정의만 유지 |
| 역할별 빠른 진입 Role Switcher | 진입 지연 | 도메인 카드 4개가 역할별 진입점 |
| 시계열 트렌드 전용 화면 | 데이터 축적 이후 의미 있음 | [D] 4주 추이 차트로 대체 |

### 기술 환경
- 최소 지원 해상도: 1280px 이상 (데스크탑 전용)
- 모바일: Phase 1 미지원
- 언어: 한국어 전용
