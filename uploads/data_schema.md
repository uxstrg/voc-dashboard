## 1. 3단 인과 구조 (프레임워크 핵심)

```
1단계: 인식 가치 (Brand Perceived Value)
    ↑ 축적
2단계: 체감 경험 (Felt Experience)
    ↑ 기반
3단계: 객관적 품질 속성 (Quality Attributes) ← VoC가 매핑되는 지점
```

VoC는 3단계(품질 속성)에 매핑되고, 그 결과가 2단계(체감 경험)를 거쳐 1단계(인식 가치)로 수렴된다.

## 2. 4대 진단 도메인 (누가 움직여야 하는가)

| 도메인 | 정의 | 핵심 담당 | 컬러 | 속성 수 |
|--------|------|-----------|------|---------|

## 3. 4대 브랜드 인식 가치 (브랜드가 어떤 타격을 입고 있는가)

> Phase 2 이후 시각화 예정. 현재는 데이터 구조 정의만 유지.

| 인식 가치 | 정의 | 연결 체감 경험 |
|-----------|------|----------------|
| 정체성 (Identity) | 브랜드 고유 철학과 특색이 느껴지는가 | 심미성, 감성, 직관성 |
| 차별성 (Differentiation) | 경쟁사 대비 새롭고 특별한가 | 적합성 |
| 유용성 (Utility) | 실질적 문제 해결에 도움이 되는가 | 경제성, 편의성 |
| 신뢰성 (Reliability) | 언제든 안심하고 이용할 수 있는가 | 안정성, 고객지원성 |

## 6. VoC 데이터 구조 (TypeScript 타입)

### 원문 VoC

```typescript
interface RawVoC {
  id: string;
  source: 'app_store_ios' | 'app_store_aos' | 'threads' | 'x' | 'community' | 'manual';
  channel_detail?: string;
  service?: string;
  raw_text: string;
  collected_at?: string;           // ISO 8601, 수집 시점에 날짜가 없으면 null
  author_context?: string;
}
```

### 진단 결과 (이슈 단위)

```typescript
interface DiagnosisIssue {
  issue_summary: string;
  feedback_type: '시각·인지' | '사용편의' | '브랜드 체감' | '감성' | '운영 품질';
  domain: '전략' | 'UX' | '운영' | '기술';
  attributes: string[];           // 29개 속성 중 1~2개
  sentiment: '매우 긍정' | '긍정' | '중립' | '부정' | '매우 부정';
  sentiment_score: number;
  severity: '일반' | '반복·우회' | '이탈·기만';
  severity_multiplier: number;
  score_reality: number;
  expectation_level: 'High' | 'Mid' | 'Standard';
  score_exp: number;
  gap: number;
  status: 'Exceeding' | 'Alignment' | 'Gap';
  diagnosis: string;
  action_hint: string;
}
```

### 진단된 VoC 전체

```typescript
interface DiagnosedVoC extends RawVoC {
  issues: DiagnosisIssue[];
  overall_summary: string;
  diagnosed_at: string;           // ISO 8601
}
```

> **사이트 투입 조건**: 사이트(`voc-dashboard`)에 하드코딩되는 데이터는 모두 진단이 완료된 `DiagnosedVoC` 형식이다. 진단 트랙에서 분석 테스트 시에는 원문(`RawVoC`) 상태의 데이터도 사용할 수 있다.
